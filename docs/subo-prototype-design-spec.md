## 2. 视觉方向

### 风格关键词

- 临床感与科研感并存
- 清洁、可信、非电商化
- 商务承接明确
- 后台偏运营工具气质

### 视觉目标

- 官网前台强调品牌与业务理解，不走泛营销风
- 报价中心保留目录检索优势，但不做客户系统
- 后台强调经营动作，不做装饰化大屏

## 4. Screen Specs

## Screen: 官网首页

### Purpose

作为公司主域名的正式入口，明确展示：

- 品牌
- 技术服务
- 试剂耗材代采
- 合作流程
- 联系方式

### Layout Structure

```text
┌──────────────────────────────────────────────┐
│ 顶部导航                                     │
├──────────────────────────────────────────────┤
│ Hero 首屏：品牌 + 两条业务线 + CTA            │
├──────────────────────────────────────────────┤
│ 技术服务能力地图                              │
├──────────────────────────────────────────────┤
│ 代采流程与平台说明                            │
├──────────────────────────────────────────────┤
│ 合作流程                                      │
├──────────────────────────────────────────────┤
│ 联系方式                                      │
└──────────────────────────────────────────────┘
```

### Key Decisions

- 首屏不直接展示后台，不再让根域名进入管理端
- 代采业务在首页占据明确位置，不再作为后台隐性能力
- 联系方式公开，开户信息不在首页直接展示

## Screen: 报价中心

### Purpose

承接两类前台线索：

- 技术服务目录报价
- 代采需求提交

### Layout Structure

```text
┌──────────────────────────────────────────────┐
│ 顶部品牌区 + 模式切换                         │
├──────────────┬──────────────────┬────────────┤
│ 筛选侧栏     │ 服务条目列表      │ 报价单摘要   │
└──────────────┴──────────────────┴────────────┘
```

### Key Decisions

- `技术服务报价` 与 `代采需求提交` 分为两个模式
- 不做客户登录
- 右侧摘要保留“已选条目 + 客户信息 + 提交询价”

## Screen: 后台仪表盘

### Purpose

作为内部经营总览，优先展示：

- 待跟进询价
- 技术服务订单
- 代采订单
- 合同 / 发票动作

### Layout Structure

```text
┌────────────┬─────────────────────────────────┐
│ 左侧导航    │ 顶部控制条                      │
│            ├─────────────────────────────────┤
│            │ 指标卡                           │
│            ├─────────────────────────────────┤
│            │ 图表 / 结构分析                  │
│            ├─────────────────────────────────┤
│            │ 最新询价 / 代采提醒              │
└────────────┴─────────────────────────────────┘
```

### Key Decisions

- 技术服务订单与代采订单必须拆开显示
- 线索来源统一汇总
- `生成清单` 的前置提醒应在首页可见

## Screen: 供应商采购清单

### Purpose

保留旧后台最高价值动作：从供应商链接快速生成采购清单。

### Layout Structure

```text
┌────────────┬─────────────────────────┬──────────┐
│ 左侧导航    │ 平台商品表格            │ 右侧清单栏 │
└────────────┴─────────────────────────┴──────────┘
```

### Key Decisions

- 锐竟 / 喀斯玛合并为平台筛选
- 商品表格与右侧清单摘要并存
- 导出、备注、关联询价作为清单参数放在右栏

## 5. 响应式原则

- 1180px 以下：三栏结构降为上下结构
- 760px 以下：卡片与统计区切为单列
- 后台在平板以下采用纵向栈式布局

## 6. Accessibility

- 交互按钮高度不低于 42px
- 重点控件采用高对比强调色
- 表格行高和信息密度平衡，不走极端紧凑

## 7. Developer Handoff

这版原型适合作为正式项目骨架前的视觉基线。下一步开发时可直接抽取：

- 颜色变量
- 页面结构
- 模块边界
- 前后台信息架构
- 报价与采购清单的关键交互
