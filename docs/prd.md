# 广州溯博生物科技有限公司官网与后台一期 PRD

## 1. 项目背景

广州溯博生物科技有限公司当前已有两套线上系统：

- 旧后台：`https://www.subo.work/dashboard`
- 报价系统：`https://subo-quote-h5.netlify.app/`

现状问题：

- 主域名没有承载官网，访问根域名会直接进入后台，不符合公司品牌展示与获客需求。
- 报价系统和后台系统割裂，客户询价、销售跟进、订单管理之间没有统一闭环。
- 旧后台模块命名和业务边界不清晰，`技术服务` 与 `试剂耗材代采` 混在同一套订单模型中。
- `锐竟/喀斯玛链接管理` 有业务价值，但被拆成两个并列模块，扩展性不足。
- 服务报价目录已经成型，但品牌包装、内容运营、数据管理和销售承接能力不足。

本项目目标是在保留核心业务能力与历史数据的前提下，建设统一的官网与后台平台。

## 2. 项目目标

### 2.1 业务目标

- 建立正式的公司官网，承载品牌展示、服务介绍、合作流程和商务联系。
- 将现有报价系统整合为官网内的 `报价中心`，统一品牌、域名和数据管理。
- 同时展示两条业务线：
  - 生物技术服务
  - 试剂耗材代采
- 建立从 `官网访问 -> 报价提交 -> 销售跟进 -> 订单管理 -> 合同与发票管理` 的业务闭环。
- 保留并升级 `锐竟/喀斯玛生成清单` 能力，用于代采业务的采购清单输出。
- 将旧后台核心数据迁移到新系统。

### 2.2 产品目标

- 官网具备良好的品牌感、可信度和移动端体验。
- 报价中心支持客户自助检索服务项目、加入报价单并提交询价。
- 后台支持内容管理、服务目录管理、线索管理、订单管理、客户管理、合同管理、供应商链接管理与权限管理。
- 一期不做客户登录，仅支持 `提交询价`。

## 3. 已确认决策

- 官网需要展示 `试剂耗材代采` 业务。
- 报价中心只支持 `提交询价`，不做客户登录和历史报价查询。
- 旧后台数据需要迁移。
- 数据库使用 `MySQL`。
- `锐竟/喀斯玛` 的 `生成清单` 功能必须保留。

## 4. 用户角色

### 4.1 外部用户

- 科研客户：浏览公司信息、技术服务、代采能力，提交询价或留言。
- 商务合作方：了解合作模式、业务范围、联系方式。

### 4.2 内部用户

- 超级管理员：负责系统配置、账号权限、全局数据管理。
- 销售/商务：跟进询价、创建订单、维护客户、生成采购清单。
- 运营：维护官网内容、服务目录、供应商链接、留言处理。
- 财务：维护发票抬头、核对回款、下载合同资料。

## 6. 功能需求

### 6.1 官网首页

#### 目标

- 对外展示公司品牌与业务范围
- 引导客户进入报价中心或直接联系

#### 核心内容

- 品牌主视觉与公司简介
- 核心业务介绍
- 技术服务分类概览
- 试剂耗材代采介绍
- 合作流程
- 常见问题
- 联系方式
- CTA：`立即询价`、`查看服务报价`

### 6.2 技术服务页面

#### 目标

- 结构化呈现生物技术服务能力

#### 核心内容

- 分类导航
- 服务简介
- 典型检测/实验方向
- 流程说明
- 交付说明
- 引导进入报价中心

### 6.3 试剂耗材代采页面

#### 目标

- 展示代采业务能力与平台资源

#### 核心内容

- 可代采的品类说明
- 采购平台说明
- 采购流程
- 供货与开票说明
- 代采询价入口

### 6.4 报价中心

#### 目标

- 提供客户自助筛选、组合和提交询价能力

#### 需求

- 展示服务分类、项目、条目、规格和价格
- 支持关键词搜索
- 支持分类筛选
- 支持细分项目筛选
- 支持价格区间筛选
- 支持加入报价单
- 支持调整条目数量
- 支持填写：
  - 姓名
  - 单位
  - 联系方式
  - 备注
- 支持提交询价
- 支持复制询价内容

#### 说明

- 报价中心仅做 `提交询价`
- 不做客户登录
- 不做在线支付
- 不做历史记录查询

### 6.5 留言与线索管理

#### 来源

- 官网联系表单
- 报价中心提交询价

#### 需求

- 统一进入后台线索池
- 支持状态流转：
  - 待跟进
  - 跟进中
  - 已转订单
  - 已关闭
- 支持按来源区分：
  - 官网留言
  - 技术服务询价
  - 代采询价

### 6.6 订单管理

#### 目标

- 将历史混合订单拆分为清晰业务模型

#### 订单类型

- 技术服务订单
- 试剂耗材代采订单

#### 核心字段

- 订单编号
- 订单类型
- 客户信息
- 项目名称
- 项目内容
- 金额
- 是否有合同
- 是否已收款
- 是否有送货单
- 下单日期
- 发票抬头
- 关联询价
- 备注

### 6.7 供应商链接管理

#### 背景

旧系统存在两个平行模块：

- 锐竟链接管理
- 喀斯玛链接管理

新系统应合并为统一模块 `供应商链接管理`。

#### 需求

- 支持平台字段：
  - 锐竟
  - 喀斯玛
  - 其他
- 支持维护产品名称、货号、类型、单位、规格、单价、采购链接、图片
- 支持搜索、筛选、批量选择
- 支持 `生成清单`
- 支持导出 Excel
- 支持将清单关联到订单或询价

### 6.8 合同管理

#### 需求

- 上传合同文件
- 下载合同文件
- 关联客户/订单
- 记录上传时间与上传人
- 支持版本备注

### 6.9 发票抬头管理

#### 需求

- 维护单位名称
- 税号
- 地址
- 电话
- 开户银行
- 银行账号
- 关联客户

### 6.10 官网内容管理

#### 需求

- 维护首页核心文案
- 维护关于我们内容
- 维护联系方式
- 维护业务介绍模块
- 维护 FAQ
- 维护 Banner/图片资源

## 7. 非功能需求

### 7.1 性能

- 官网首屏加载要轻量，移动端可用
- 报价目录支持 800+ 条以上项目顺畅筛选
- 后台列表页支持分页、检索与导出

### 7.2 SEO

- 官网需支持 SEO 基础能力
- 页面支持独立标题、描述和分享图

### 7.3 安全

- 后台必须独立子域名
- 后台接口需鉴权
- 上传文件需限制类型和大小
- 敏感公司信息不在官网首页直接公开展示银行账户

### 7.4 可维护性

- 代码应支持后续扩展二期功能
- 服务目录与官网内容应后台可维护，尽量不依赖手工改代码

## 8. 验收标准

### 8.1 官网

- 根域名访问进入正式官网，而不是后台
- 官网可完整展示品牌、技术服务、代采业务和联系方式
- 移动端浏览正常

### 8.2 报价中心

- 客户可以检索服务并提交询价
- 询价可在后台查看
- 后台可区分询价来源

### 8.3 后台

- 可管理服务目录、询价、订单、客户、合同、发票抬头、供应商链接
- 可生成并导出采购清单

### 8.4 数据迁移

- 旧后台核心历史数据可查询
- 锐竟/喀斯玛数据已并入统一供应商模块
- 历史订单数据无关键字段丢失
